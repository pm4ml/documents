Starting up the Mojaloop Connector SDK Scheme Adapter

To start up the Mojaloop Connector SDK Scheme Adapter, complete the following steps.

Prerequisites:

  • Ensure you have Docker installed in your local environment.

  • Familiarity with Docker is advised.

Steps:

  1. Clone the Mojaloop Connector SDK Scheme Adapter repository locally:

    1. Change directory into your workspace, then to clone using HTTPS:

      $ git clone https://github.com/modusintegration/mojaloop-connector

      Or clone using SSH:

      $ git clone git@github.com:modusintegration/mojaloop-connector.git

    2. Now change directory into the cloned repository directory:

      $ cd mojaloop-connector

  2. Make a copy of the file .env.example (to be found in the src folder) and rename your copy to .env. Make your changes as necessary. For details on Sheme Adapter configuration options, see xref:???.adoc[Configuring the Mojaloop Connector SDK (Scheme Adapter)].

  3. Start up the container using docker run, specifying .env as the environment file:

    $ docker run --env-file src/.env modusintegration/mojaloop-connector

    This starts up the Mojaloop Connector SDK Scheme Adapter without the Redis cache.

Note that it is possible to run the Scheme Adapter without Docker, which can be useful during development. To run the Scheme Adapter without Docker, issue the following commands in succession:

$ cd mojaloop-connector/src
$ npm install
$ cp .env.example .env # modify .env accordingly
$ node index.js

When using this method, the Scheme Adapter picks up the .env file automatically.
