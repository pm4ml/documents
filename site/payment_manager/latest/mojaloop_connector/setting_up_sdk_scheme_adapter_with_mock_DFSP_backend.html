Setting up the Mojaloop Connector SDK Scheme Adapter with a mock DFSP backend

This section provides quick start guidance to setting up the Mojaloop Connector SDK Scheme Adapter locally with a mock DFSP backend. This configuration is suitable as a starting point for DFSPs wishing to utilise the Mojaloop Connector SDK Scheme Adapter for integrating their backend systems with a Mojaloop-API-enabled Hub.

The instructions provided here are for Linux-based systems. For Mac and/or Windows, you will need to translate the following for your environment.

Prerequisites:

Make sure you have docker and docker-compose installed locally. For detailed instructions, see the Install Docker and Install Docker Compose documentation by Docker, respectively.

Steps:

  1. Clone the Mock DFSP Backend repository locally:

    1. Change directory into your workspace, then to clone using HTTPS:

      $ git clone https://github.com/modusbox/mojaloop-sdk-mock-dfsp-backend.git

      Or clone using SSH:

      $ git clone git@github.com:modusbox/mojaloop-sdk-mock-dfsp-backend.git

    2. Now change directory into the cloned repository directory:

      $ cd mojaloop-sdk-mock-dfsp-backend

  2. Edit the Scheme Adapter configuration to point the Scheme Adapter at your Mojaloop-API-enabled Hub or simulator:

    1. Use your favourite text editor to edit the file src/scheme-adapter.env. Change the value of the PEER_ENDPOINT variable to the DNS name or IP address and port number of your Mojaloop-API-enabled Hub or simulator.

    2. Save the file.

  3. Use docker-compose to download and run the pre-built Scheme Adapter, shared cache (redis) and mock DFSP backend containers locally. Change directory into the src subfolder and run docker-compose:

    $ cd src/
$ docker-compose up

    You should see docker download the pre-built docker images for the Scheme Adapter, shared cache (redis), and mock DFSP backend. Docker-compose will start the containers.

  4. Test the outbound (sending money) API:

    1. Find the IP address of the mock DFSP backend container. To do this, find the list of docker networks on your local machine:

      $ docker network ls

    2. Identify the docker network created by docker-compose. Docker-compose will assign a name based on the directory name from which you ran the docker-compose up command.

    3. Once you have identified the network, you can use:

      $ docker network inspect {network name}

      This will print a JSON structure to the terminal containing the set of containers in the network and their individual IP addresses.

    4. Use the following command to tell the mock DFSP backend to initiate an outbound money transfer via the Scheme Adapter:

      NOTE: Do not forget to substitute in the correct IP address for the mock DFSP backend container.

      curl -X POST \
  http://{MOCK DFSP Backend Container IP Address}:3000/send \
  -H 'Content-Type: application/json' \
  -d '{
    "from": {
        "displayName": "John Doe",
        "idType": "MSISDN",
        "idValue": "123456789"
    },
    "to": {
        "idType": "MSISDN",
        "idValue": "987654321"
    },
    "amountType": "SEND",
    "currency": "USD",
    "amount": "100",
    "transactionType": "TRANSFER",
    "note": "test payment",
    "homeTransactionId": "123ABC",
    "quoteRequestExtensions": [
        {
            "key": "KYCNationality",
            "value": "US"
        }
    ]
}'

      The POST /send call forwards to POST /transfers in the Mojaloop Connector SDK Scheme Adapter. The response from the above call should indicate the result of the communication between the Mojaloop Connector SDK Scheme Adapter and the Mojaloop-API-enabled Hub or simulator.

  5. Speak to your Hub Operator or use your simulator to test the inbound (receiving money) API.

  6. You can now examine the code of the mock DFSP backend to understand how it implements the Mojaloop Connector SDK Scheme Adapter simplified inbound API.
